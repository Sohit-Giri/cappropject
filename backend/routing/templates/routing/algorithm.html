⚙ Dijkstra's Algorithm

Step-by-step interactive explainer of the shortest path algorithm.
How Dijkstra Works
  1. Start at the source node — assign distance 0. All others get ∞.
  2. Add source to a min-priority queue.
  3. Pop the node with smallest known distance.
  4. For each neighbour: calculate distance through current node.
  5. If shorter than known distance → update and re-queue.
  6. Repeat until the destination is popped from the queue.
  7. Trace back via predecessor pointers to build the path.
Complexity
O((V + E) log V) — with a min-heap priority queue
V = vertices (nodes), E = edges. For Nepal 5-district graph: ~50K nodes, ~120K edges.
